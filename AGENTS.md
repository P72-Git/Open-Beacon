# Antigravity Agent Instructions: Open Beacon

## Project Vision & Core Directives
**Primary Domain:** GeoDev & Spatial Data 🗺️, PostGIS 🌍, Spatial databases, and geographic workflows 📌.

When an Antigravity agent assumes control of or iterates on this project, they must adhere to the following architectural and functional goals:

1. **The Ultimate Goal (Evolutionary Architecture):**
   While this project currently operates as a local search app using React, Leaflet, and the Gemini API for mapping and data grounding, the *true goal* is to evolve into a robust spatial platform. Future iterations must prioritize implementing or interfacing with real spatial databases (like PostGIS) and complex geographic workflows.
   
2. **Geospatial Rigor:**
   Always prioritize geographic accuracy. When building features, evaluating external APIs, or processing spatial data, apply GeoDev and geospatial analysis best practices. Treat locations as verifiable geospatial entities (Points, Polygons, Geometries) to enable true spatial querying, rather than simple isolated lat/lng strings.

3. **Location-Based Application Focus:**
   Every feature built should enhance mapping, navigation, or spatial context. The user interface must remain frictionless while hiding complex geospatial querying (like distance calculating, bounding box searches, and route mapping) behind simple interactions.

4. **Current vs. Target Stack:**
   - **Current Setup:** React, Vite, Tailwind CSS, Leaflet, Gemini API (Google Search/Maps grounding), Firebase (Auth/Firestore).
   - **Target Trajectory:** Integrations with PostGIS, advanced spatial queries, specialized routing logic, and comprehensive geographic analysis layers.

*Agent Note: Do not merely document or maintain the current state. Read the user's intent as a mandate to drive the codebase toward a true spatial database and GeoDev workflow. Every backend expansion should consider spatial data viability.*
